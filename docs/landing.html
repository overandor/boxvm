What BoxVM does

📦

Pack

Compress any directory of HTML, CSS, JS, WASM into a single .box file with real zlib compression and SHA-256 integrity hashing.

▶️

Run

Serve .box containers via C++ HTTP server. Browser loads files into a Service Worker VFS. WASM binaries execute via real WASI.

🔀

Split

Agents running inside containers can split them into sub-containers with isolated VFS state. Hot-reload files without restart.

⚡

Parallel

C++ work-stealing thread pool for packing. Web Workers for parallel browser decompression. SharedArrayBuffer for WASM threads.

🔌

Agent API

WebSocket protocol for live container control. Add, remove, or patch files at runtime. Broadcast events to connected clients.

🌐

Public URL

Expose local containers with --public flag. Random or custom subdomain. Share with anyone, no install required.

Quick Start

# Build BoxVM from source make # Pack a directory into a .box container ./boxpack ./my-website output.box --parallel # Run it locally ./boxrun output.box 7860 # Run with a public URL ./boxrun output.box 7860 --public myapp # Open in browser → xterm.js terminal + file browser + agent panel # Click .wasm files to execute real binaries via WASI
