Features

Everything Docker does,
in the browser

Pack, run, split, and orchestrate containers without a single daemon or cloud dependency.

📦

Pack

Compress any directory of HTML, CSS, JS, WASM into a single .box file with real zlib compression and SHA-256 integrity hashing per file, manifest, and trailer.

zlib + OpenSSL
▶️

Run

Serve .box containers via C++ HTTP server. Browser loads files into a Service Worker VFS. WASM binaries execute via real WASI with filesystem access.

WASI + SW VFS
🔀

Split

Agents running inside containers can split them into sub-containers with isolated VFS state. Hot-reload files without restart. Inter-container IPC via SharedArrayBuffer.

Agent-driven
⚡

Parallel

C++ work-stealing thread pool for packing. Web Workers for parallel browser decompression across all CPU cores. SharedArrayBuffer for WASM threads.

Work-stealing
🔌

Agent API

WebSocket protocol (RFC 6455) for live container control. Add, remove, or patch files at runtime. Broadcast events to all connected clients.

RFC 6455
🌐

Public URL

Expose local containers with --public flag. Random or custom subdomain. Share with anyone — no install, no Docker, no cloud account required.

Zero-config
🧠

RL Optimizer

Q-learning agent learns optimal compression strategies per file type. 200-state Q-table, epsilon-greedy exploration, live reward curve visualization.

Q-Learning
💬

LLM Intelligence

Abstract LLM analysis layer. Natural language queries about container contents. Pattern detection, redundancy analysis, anomaly flagging, compression suggestions.

Container AI
📊

Entropy Analysis

Per-file Shannon entropy measurement. Byte-level histogram analysis. Identifies high-entropy files that resist compression and low-entropy files with maximum compression potential.

Shannon
🔍

Anomaly Detection

Statistical outlier detection on file size and entropy distributions. Flags unusual files that deviate significantly from container norms for security and optimization.

Statistical
📥

Upload & Inspect

Drag any .box file into the portal. Hex inspector shows raw binary structure. Metadata table displays magic, version, flags, manifest hash, and trailer hash.

Binary
📤

Export

Download the loaded container as a .box file. Works with both server-fetched and demo-generated containers. Full round-trip: pack → load → inspect → export.

Round-trip
📈

Compression Predictor

ML model predicts compression ratio from entropy + file type features. Per-file predictions with model accuracy scoring against actual ratios.

Predictive

Quick Start

Three commands to run

No install. No daemon. No cloud account.

terminal
# Clone and build git clone https://github.com/overandor/boxvm.git cd boxvm && make # Pack a directory into a .box container ./boxpack ./my-website output.box --parallel # Run it — opens in any browser ./boxrun output.box --public myapp # Browser opens → xterm.js terminal + file browser + agent panel # Click .wasm files to execute real binaries via WASI

Roadmap

From here to production

Six weeks to v1.0. Three months to ML-optimized v2.0.

v0.3 — Real Core

Completed

zlib compression, SHA-256 integrity, work-stealing thread pool, Rust→WASM compilation, WebSocket agent, xterm.js terminal

v0.4 — WASI Filesystem

Completed

Full WASI with path_open, fd_read, fd_seek, fd_readdir. WASM programs can read files from container VFS.

v0.5 — Landing Page + Developer Portal

Completed

Investor-grade landing page, interactive developer portal with xterm.js, API documentation, file browser, entropy analysis, RL compression optimizer, LLM container intelligence, anomaly detection

v0.6 — GitHub Pages + Public URL

Completed

Live on GitHub Pages. Demo container generator for browser-only mode. Relative paths for subpath hosting. Public URL sharing via --public flag.

v0.7 — Upload, Inspect, Predict

Completed

Upload .box files directly. Hex inspector with ASCII sidebar. Container metadata table. Compression ratio predictor with model accuracy scoring. Export/download loaded containers.

v0.8 — WASI Threads

In Progress

wasm32-wasip1-threads, SharedArrayBuffer between Web Workers, real parallel execution inside containers

v0.9 — Agent Framework

4 weeks

Agent SDK for in-container programs. Read/write VFS, split containers, inter-container IPC

v1.0 — Production

6 weeks

TLS, auth, IndexedDB persistence, checkpointing, CI/CD, benchmark vs Docker cold start

v2.0 — ML Optimization

3 months

ONNX Runtime for real model inference, KV cache for layout decisions, gradient-based compression, cross-validation
